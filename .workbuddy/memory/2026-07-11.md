## 关键环境坑（最重要，下次直接复用）
本机在 Clash 代理后，Android Gradle 构建踩了 4 个连环坑：

1. **Clash 代理端口是动态的**（环境变量 HTTPS_PROXY 在会话间变化：59535→7890→59824）。
   → 不要在 `gradle.properties` 写死代理端口。让 Gradle 自动读 shell 的 `HTTPS_PROXY` 环境变量即可。

2. **Clash 对 Gradle/Java 的 TLS 1.3 ClientHello 做重置** → "Remote host terminated the handshake"。
   → 在 `android/gradle.properties` 的 `org.gradle.jvmargs` 里强制 `-Djdk.tls.client.protocols=TLSv1.2`（已写入，勿删）。

3. **build-tools;35.0.0 / platforms;android-35 经代理拉到损坏包**，留下空壳目录导致 Gradle 反复重装失败。
   → 用 `sdkmanager` + 环境变量代理(读 HTTPS_PROXY) + `JAVA_OPTS=-Djdk.tls.client.protocols=TLSv1.2` 干净重装。当前已装好 build-tools 35.0.0 / platform 35 / platform-tools / NDK 28.2.13676358。

4. **Go 模块经默认 `proxy.golang.org` 拿 Bad Gateway**（Clash 国外上游不通）。
   → 构建时设 `GOPROXY=https://goproxy.cn,direct` + `GOSUMDB=off`。

## 可靠构建命令（已验证可跑通）
```bash
cd /Users/psd/Documents/dabuliuai
export HTTPS_PROXY=http://127.0.0.1:$(端口)   # 从当前 env 取，勿写死
export HTTP_PROXY=$HTTPS_PROXY
export GOPROXY=https://goproxy.cn,direct
export GOSUMDB=off
export GOMAXPROCS=2                         # 限制 Go CGO 并发，降资源峰值
export ANDROID_NDK=/Users/psd/Library/Android/sdk/ndk/28.2.13676358
export JAVA_HOME=$(/usr/libexec/java_home)
.tooling/flutter/bin/flutter build apk --dart-define-from-file=env.json --split-per-abi --release
```
- Flutter 在 `.tooling/flutter`（3.44.6，Dart 3.12.2）；Go 1.26.2；Java Temurin 24（路径由 /usr/libexec/java_home 给）。
- 本机内存充裕（~76GB free），**不是 OOM**。构建若卡死多为代理/TLS/SDK 组件损坏，按上面 4 点排查。
- 后台托管任务（run_in_background）可稳定存活 5+ 分钟，无时间墙。普通 Bash 调用的子进程在调用返回时被清掉（setsid/disown 也无效），长任务务必用 run_in_background。
- 日志管道 `flutter ... 2>&1 | while read; do ...; done | tee` 会把 Gradle 的 `\r` 进度吞掉（只认 `\n`），构建其实在跑、只是不刷行，等阶段结束刷 `\n` 才显示——别误判为卡死。
